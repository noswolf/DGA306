บทนำและขอบเขตวิทยาการข้อมูล (Data Science Overview)

สรุปเนื้อหาแกนหลักประจำชุดการเรียนรู้ เพื่อทำความเข้าใจความเกี่ยวเนื่องของโมเดลการวิเคราะห์

เส้นทางการเรียนรู้: จากจัดเก็บข้อมูลสู่วิทยาการทำนายอนาคต

ใน Day 1-3 คุณได้ผ่านกระบวนการคัดกรองข้อมูล ปูพื้นฐานการควบคุมด้วย Python และจัดเก็บข้อมูลอย่างเป็นระเบียบลงคลังสินค้าข้อมูล (Data Warehouse) แล้ว ซึ่งนั่นคือรากฐานข้อมูลประวัติศาสตร์ (Descriptive & Diagnostic) สำหรับใน Day 4 นี้ เราจะยกระดับขึ้นสู่การพยากรณ์ด้วยปัญญาประดิษฐ์ (Predictive Analytics) เพื่อให้โมเดลจำลองเรียนรู้จากอดีตและทำนายผลลัพธ์ในอนาคตได้อย่างอัตโนมัติ

Data Science คืออะไร?

วิทยาการข้อมูลเกิดจากการบูรณาการความรู้ของ 3 ศาสตร์หลัก ได้แก่ Computer Science (ทักษะไอที/การเขียนโปรแกรม), Math & Statistics (คณิตศาสตร์และการคำนวณโมเดล) และ Business Domain Expertise (ความรู้ความเข้าใจในบริบทของธุรกิจหน้างาน)

4 ระดับขั้นของการวิเคราะห์ข้อมูล (4 Types of Data Analytics)

1. Descriptive Analytics

บอกว่าเกิดอะไรขึ้นในอดีต (What happened?) เช่น รายงานสรุปยอดขายประจำปี

2. Diagnostic Analytics

เจาะลึกหาสาเหตุว่าทำไมจึงเกิดขึ้น (Why did it happen?) เช่น การทำ Data Drill-Down

3. Predictive Analytics (เน้นใน Day 4)

พยากรณ์สิ่งที่มีแนวโน้มจะเกิดขึ้นในอนาคต (What will happen?) โดยอาศัยเทคนิคทางสถิติและ Machine Learning

4. Prescriptive Analytics

แนะนำทางเลือกและผลลัพธ์ที่ควรทำ (How can we make it happen?) เพื่อช่วยชี้นำการตัดสินใจสูงสุด

ทบทวนงานของ Supervised Learning (Regression vs Classification)

Regression (การทำนายค่าต่อเนื่อง): ผลลัพธ์เป็นค่าตัวเลขเชิงปริมาณที่มีสเกลต่อเนื่อง มุ่งเน้นหาแนวโน้มและความสัมพันธ์เชิงเส้นตรงหรือเส้นโค้งเพื่อทำนายสเกลตัวเลข เช่น รายได้ หรืออุณหภูมิ
Classification (การจำแนกประเภทกลุ่ม): ผลลัพธ์เป็นกลุ่มหรือหมวดหมู่ประเภทที่ตัดขาดออกจากกันชัดเจน (Discrete Class) มุ่งเน้นหาเส้นแบ่งเขตเพื่อแยกประเภทข้อมูลออกจากกัน เช่น ผ่าน/ไม่ผ่าน หรือ ป่วย/ปกติ

Linear Regression Simulation

ทดลองปรับค่าสัมประสิทธิ์ความชัน (Slope) และจุดตัดแกน Y (Intercept) เพื่อสังเกตกลไกการคำนวณค่าความคลาดเคลื่อนสะสม

แนวคิดหัวใจสำคัญ: Linear Regression คืออะไร?

คือโมเดลการพยากรณ์ข้อมูลที่เป็น ตัวเลขต่อเนื่องเชิงปริมาณ (เช่น ยอดขายรายได้ หรืออุณหภูมิ) โดยหาความสัมพันธ์ระหว่างปัจจัยอิสระ $x$ (ตัวแปรต้น) กับผลลัพธ์ $y$ (ตัวแปรตาม) ออกมาเป็นสมการเส้นตรง:

ŷ = β₀ + β₁x
β₀ (Intercept / จุดตัดแกน Y) ค่าเริ่มต้นทำนายเมื่อแกน X เป็น 0 (เช่น แม้ไม่ลงโฆษณาเลย ก็มียอดขายเริ่มต้นที่ β₀ บาท)
β₁ (Slope / ความชัน) อัตราการเติบโตเมื่อ X เพิ่มขึ้น 1 หน่วย (เช่น ลงทุนโฆษณาเพิ่ม 1 บาท ยอดขายจะขึ้นอีก β₁ บาท)
🎯 ความคลาดเคลื่อนกำลังสอง (SSE): ระยะห่างความเพี้ยนระหว่างจุดจริงกับเส้นทำนายจะถูกจับมา ยกกำลังสอง แล้วรวมกัน โมเดลที่ดีที่สุดคือตัวแปรที่สามารถคำนวณปรับแต่งความชันเพื่อลดค่า SSE ให้ต่ำที่สุดเท่าที่จะทำได้!

คอนโทรลพารามิเตอร์

สมการทำนาย: ŷ = 20 + (0.5)x
Sum of Squared Errors (SSE) - ผลรวมความคลาดเคลื่อนยกกำลังสอง 0.00

หลักการทำงาน: เป้าหมายของโมเดลคือการปรับค่า β₀ และ β₁ เพื่อให้เส้นพยากรณ์ลากผ่านใกล้จุดข้อมูลจริงมากที่สุด ซึ่งจะทำให้ค่าความคลาดเคลื่อนยกกำลังสอง (SSE) มีค่าต่ำที่สุดตามหลักการคณิตศาสตร์ Least Squares Method

เส้นประสีเขียวแสดง "Residuals" (ระยะห่างความเพี้ยนข้อมูล) ระหว่างค่าจริง (จุดแดง) กับเส้นทำนาย (สีน้ำเงิน)

Classification & Logistic Regression

เบื้องหลังการทำนายกลุ่มมาจากอะไร? โมเดลจะนำฟีเจอร์คำนวณผ่านฟังก์ชันคณิตศาสตร์เพื่อแปลงค่าเป็นระดับความน่าจะเป็นแบบ S-Curve

แนวคิดหัวใจสำคัญ: Logistic Regression กับการจำแนกกลุ่ม

ใช้ทำนายผลลัพธ์ที่เป็น กลุ่ม/หมวดหมู่ประเภทขาดจากกัน (เช่น ป่วยอ้วนจริง=1, คนปกติ=0) สาเหตุที่เราไม่ใช้สมการเส้นตรงปกติเนื่องจากความน่าจะเป็นของโอกาสที่จะเกิด **จะต้องอยู่ระหว่าง 0% ถึง 100% (0.0 ถึง 1.0) เสมอ**

📈 1. ดัดให้โค้งมนด้วย Sigmoid (S-Curve) โมเดลจะนำผลรวมสมการเส้นตรงมาป้อนผ่านฟังก์ชันคณิตศาสตร์ปรับความโค้ง เพื่อบีบจำกัดสเกลให้อยู่กึ่งกลางระหว่าง 0 ถึง 1 พอดี
🛑 2. ตัดสินใจด้วย Decision Boundary เกณฑ์ตัดสินใจ (เช่น 0.5) หากความน่าจะเป็นสูงพ้นเกณฑ์จะปัดผลเป็นคลาส 1 ทันที หากไม่ถึงจะเป็นคลาส 0

กลไกการคำนวณแยกกลุ่ม

ขั้นตอนที่ 1: คำนวณสมการเชิงเส้นตรง (Log-odds)

z = β₀ + β₁X

ขั้นตอนที่ 2: ส่งผ่านฟังก์ชันปรับความโค้ง Sigmoid

P = 1 / (1 + e-z)

ผลลัพธ์ย่อย P คือค่าความน่าจะเป็น (Probability) ในช่วง 0 ถึง 1 (แสดงด้วยความสูงของเส้นโค้ง S-Curve บนกราฟ)

กติกาแบ่งกลุ่ม: หากจุดข้อมูลมีระดับความน่าจะเป็นลอยสูงพ้นเกณฑ์เส้นปะสีแดง โมเดลจะจัดให้เป็น Class 1 (โรคอ้วน) ถ้าไม่ถึงจะจัดเป็น Class 0 (ปกติ) จุดวงแหวนสีแดงซ้อนภายนอกหมายถึงข้อมูลเกิดพยากรณ์ผิดพลาด (Misclassification)
แกนตั้งแสดงโอกาสความน่าจะเป็น 0.0 ถึง 1.0 (0% ถึง 100%) | จุดสีฟ้าคือคนปกติ จุดสีส้มคือคนเป็นโรคอ้วนจริง (วงกลมสีแดงซ้อนนอก = ทายผิดกลุ่ม)

นำผลต่างความคลาดเคลื่อนของข้อมูลแต่ละจุดมาถอดเครื่องหมายลบออกให้กลายเป็นบวก (Absolute Value) แล้วจึงนำมาหาค่าเฉลี่ย วิธีนี้จะไม่ถูกดึงหรือกระทบโดยค่าที่หลุดกลุ่มรุนแรง (Outliers) มากเกินไป ช่วยสะท้อนขนาดความเพี้ยนเฉลี่ยในหน่วยปกติจริง

ลำดับขั้นตอนคณิตศาสตร์ (Algorithm Steps):
1. หาผลต่างย่อย: (ค่าจริง - ค่าทำนาย) ของข้อมูลในแต่ละคู่อันดับ
2. แปลงค่าสัมบูรณ์: ถอดเครื่องหมายติดลบออกให้กลายเป็นบวกทั้งหมดเพื่อคงมูลค่าดิบ
3. คำนวณหาค่าเฉลี่ย: นำผลรวมทั้งหมดมาหารด้วยจำนวนปริมาณข้อมูลรวม (N)

Confusion Matrix & Linked Sliders

ตารางตรวจสอบความแม่นยำกลุ่ม: แถบสไลเดอร์ทำงานสัมพันธ์เชื่อมโยงกันอย่างสมดุลตามสัดส่วนปริมาณประชากรจริงในระบบ

แนวคิดหัวใจสำคัญ: การประเมินผลกลุ่ม (Confusion Matrix)

ในการจำแนกกลุ่ม ความแม่นยำรวม (Accuracy) อาจหลอกตาเราได้ง่าย (เช่น คนป่วยมี 1 ใน 100 คน ถ้าทายว่าปกติทุกคนก็ได้ความแม่นยำรวมสูงถึง 99% ทันทีแต่ไร้ประโยชน์สูงสุด) ตาราง Confusion Matrix จึงช่วยจำแนกดูจุดบกพร่องแยกประเภท:

🎯 Accuracy (ความแม่นยำรวม) สัดส่วนที่ทำนายประเภทได้ถูกต้องทั้งหมด $\rightarrow$ (TP + TN) / Total
🔍 Precision (ความแม่นยำคาดเดาบวก) สัดส่วนการทายว่าบวกแล้วถูกจริง เพื่อจำกัดความตื่นตูมเตือนผิดพลาด (FP) $\rightarrow$ TP / (TP + FP)
🛡️ Recall (ความสามารถตรวจเจอ) สัดส่วนคนเป็นโรคจริงที่ถูกค้นหาเจอ เพื่อจำกัดไม่ให้หลุดรอดขาดตรวจ (FN) $\rightarrow$ TP / (TP + FN)
⚖️ F1-Score: ค่าเฉลี่ยกลางแบบฮาร์มอนิก (Harmonic Mean) เพื่อประเมินความสอดคล้องโดยไม่ปล่อยให้ค่าใดค่าหนึ่งของ Precision หรือ Recall มีค่าต่ำจนเกินไป
สถานการณ์คัดกรองโรคระบาด (กำหนดกลุ่มประชากรรวมคงที่ 200 รายชิ้น)

• กลุ่มผู้ติดเชื้อจริง (Actual Positive) = บังคับคงที่ 50 ราย (คำนวณแชร์สัดส่วนระหว่าง ทายถูกตรงเป๊ะ กับ ทำหลุดรอดไป)

• กลุ่มคนปกติไม่ติดเชื้อจริง (Actual Negative) = บังคับคงที่ 150 ราย (คำนวณแชร์สัดส่วนระหว่าง ทายตื่นตูมฟรี กับ ทายถูกว่าปกติ)
