2 ประเภทหลักของ Machine Learning

สิ่งที่ชี้ขาดว่าจะใช้โมเดลแบบไหน คือข้อมูลที่เรามี "มีผลเฉลย (Label)" ติดมาด้วยหรือไม่

Supervised Learning

เรียนรู้แบบ "มีผู้สอน"

เรียนรู้จากข้อมูลที่ มีผลเฉลย (Labeled Data) กำกับไว้ครบทุกแถว อัลกอริทึมจะค่อย ๆ เรียนรู้ความสัมพันธ์ระหว่างคุณลักษณะ (Feature) กับผลเฉลย (Target) เพื่อนำไปทำนายข้อมูลใหม่ที่ยังไม่เคยเห็น

📥 ข้อมูลนำเข้า Feature + Target (เฉลย) เช่น [อุณหภูมิ → จำนวนจักรยานที่ถูกเช่า]
Regression ทำนายค่าต่อเนื่อง
Classification ทำนายหมวดหมู่

Unsupervised Learning

เรียนรู้แบบ "ไม่มีผู้สอน"

เรียนรู้จากข้อมูลที่ ไม่มีผลเฉลย (Unlabeled Data) มีเพียงคุณลักษณะ (Feature) เท่านั้น อัลกอริทึมจะค้นหา รูปแบบ (Pattern) หรือโครงสร้างที่ซ่อนอยู่ภายในข้อมูลด้วยตัวเอง โดยไม่มีคำตอบที่ถูกต้องคอยบอก

📥 ข้อมูลนำเข้า Feature เพียงอย่างเดียว (ไม่มีเฉลย) เช่น [ปริมาณการปล่อยมลพิษของแต่ละพื้นที่]
Clustering จับกลุ่มข้อมูล
Dimensionality Reduction ลดมิติข้อมูล (PCA)
คลิกเลือกการ์ด Supervised หรือ Unsupervised ด้านบนเพื่อสลับดูเนื้อหาเปรียบเทียบ

Linear Regression Simulation

ทดลองปรับค่าสัมประสิทธิ์ความชัน (Slope) และจุดตัดแกน Y (Intercept) เพื่อสังเกตกลไกการคำนวณค่าความคลาดเคลื่อนสะสม

แนวคิดหัวใจสำคัญ: Linear Regression คืออะไร?

คือโมเดลการพยากรณ์ข้อมูลที่เป็น ตัวเลขต่อเนื่องเชิงปริมาณ (เช่น ยอดขายรายได้ หรืออุณหภูมิ) โดยหาความสัมพันธ์ระหว่างปัจจัยอิสระ $x$ (ตัวแปรต้น) กับผลลัพธ์ $y$ (ตัวแปรตาม) ออกมาเป็นสมการเส้นตรง:

ŷ = β₀ + β₁x
β₀ (Intercept / จุดตัดแกน Y) ค่าเริ่มต้นทำนายเมื่อแกน X เป็น 0 (เช่น แม้ไม่ลงโฆษณาเลย ก็มียอดขายเริ่มต้นที่ β₀ บาท)
β₁ (Slope / ความชัน) อัตราการเติบโตเมื่อ X เพิ่มขึ้น 1 หน่วย (เช่น ลงทุนโฆษณาเพิ่ม 1 บาท ยอดขายจะขึ้นอีก β₁ บาท)
🎯 ความคลาดเคลื่อนกำลังสอง (SSE): ระยะห่างความเพี้ยนระหว่างจุดจริงกับเส้นทำนายจะถูกจับมา ยกกำลังสอง แล้วรวมกัน โมเดลที่ดีที่สุดคือตัวแปรที่สามารถคำนวณปรับแต่งความชันเพื่อลดค่า SSE ให้ต่ำที่สุดเท่าที่จะทำได้!

คอนโทรลพารามิเตอร์

สมการทำนาย: ŷ = 20 + (0.5)x
Sum of Squared Errors (SSE) - ผลรวมความคลาดเคลื่อนยกกำลังสอง 0.00

หลักการทำงาน: เป้าหมายของโมเดลคือการปรับค่า β₀ และ β₁ เพื่อให้เส้นพยากรณ์ลากผ่านใกล้จุดข้อมูลจริงมากที่สุด ซึ่งจะทำให้ค่าความคลาดเคลื่อนยกกำลังสอง (SSE) มีค่าต่ำที่สุดตามหลักการคณิตศาสตร์ Least Squares Method

เส้นประสีเขียวแสดง "Residuals" (ระยะห่างความเพี้ยนข้อมูล) ระหว่างค่าจริง (จุดแดง) กับเส้นทำนาย (สีน้ำเงิน)

Classification & Logistic Regression

หัวใจสำคัญของการทำนายกลุ่มและการตัดสินใจอย่างมีหลักการ

แนวคิดหลัก: Logistic Regression คืออะไร?

Classification Fundamentals

Logistic Regression เป็นอัลกอริทึมที่ใช้สำหรับ Classification (การแยกประเภท) โดยจะคำนวณหา ความน่าจะเป็น (Probability) ที่ข้อมูลจะอยู่ในกลุ่มนั้น ๆ ก่อนตัดสินใจเลือกผลลัพธ์สุดท้าย

ใช้กับ Classification แม้ชื่อจะมีคำว่า "Regression" แต่ผลลัพธ์สุดท้ายเป็น "กลุ่ม" (เช่น ผ่าน/ไม่ผ่าน) ไม่ใช่ค่าตัวเลขต่อเนื่อง
Sigmoid (0 ถึง 1) โมเดลจะแปลงค่าสมการให้เป็น ความน่าจะเป็น ซึ่งต้องอยู่ระหว่าง 0% ถึง 100% เสมอ จึงต้องใช้ S-Curve (Sigmoid)
Decision Boundary เกณฑ์ตัดสินใจ (Threshold) เช่น ถ้าโอกาสเกิน 0.5 ให้จัดเป็นคลาส 1 ทันที เพื่อระบุคลาสผลลัพธ์สุดท้าย
ทำไมชื่อ "Regression" แต่ใช้แยกกลุ่ม (Classification)?

เพราะเบื้องหลังคณิตศาสตร์ของมันคือการหา "ตัวเลขความน่าจะเป็น" (เลขต่อเนื่องเหมือน Regression)
แต่เป้าหมายสุดท้ายคือการนำตัวเลขนั้นมา "ปัดเศษเลือกกลุ่ม" มันจึงเป็นโมเดลที่ใช้สำหรับ Classification นั่นเองครับ

💡 ลองนึกภาพ: เหมือนเครื่องวัดอุณหภูมิที่วัดได้ 38.5 องศา (ตัวเลข = Regression) แต่เราใช้ตัวเลขนี้เพื่อตัดสินใจว่าคนนี้ "ป่วย" หรือ "ปกติ" (เลือกกลุ่ม = Classification)

กลไกการคำนวณแยกกลุ่ม

1. คำนวณสมการเชิงเส้นตรง (z)

z = β₀ + β₁X

2. ส่งผ่าน Sigmoid Function

P = 1 / (1 + e-z)

ผลลัพธ์ P คือความสูงของเส้นโค้ง S-Curve บนกราฟ (0 ถึง 1)

กติกาการแบ่งกลุ่ม: จุดที่มีระดับความน่าจะเป็นลอยสูงพ้นเกณฑ์เส้นประสีแดง จะจัดเป็น Class 1 (โรคอ้วน) ถ้าต่ำกว่าจะเป็น Class 0 (ปกติ)
* วงแหวนสีแดงรอบจุด = ข้อมูลที่ทำนายผิดพลาด (Misclassification)

คำอธิบายกราฟ: แกนตั้งแสดงความน่าจะเป็น 0.0 - 1.0 (0% - 100%)
จุดสีฟ้า = คนปกติ | จุดสีส้ม = คนเป็นโรคอ้วนจริง

ลำดับขั้นตอนคณิตศาสตร์ (Algorithm Steps):
1. หาผลต่างย่อย: (ค่าจริง - ค่าทำนาย) ของข้อมูลในแต่ละคู่อันดับ
2. แปลงค่าสัมบูรณ์: ถอดเครื่องหมายติดลบออกให้กลายเป็นบวกทั้งหมดเพื่อคงมูลค่าดิบ
3. คำนวณหาค่าเฉลี่ย: นำผลรวมทั้งหมดมาหารด้วยจำนวนปริมาณข้อมูลรวม (N)

แนวคิดหัวใจสำคัญ: การประเมินผลกลุ่ม (Confusion Matrix)

ในการจำแนกกลุ่ม ความแม่นยำรวม (Accuracy) อาจหลอกตาเราได้ง่าย (เช่น คนป่วยมี 1 ใน 100 คน ถ้าทายว่าปกติทุกคนก็ได้ความแม่นยำรวมสูงถึง 99% ทันทีแต่ไร้ประโยชน์สูงสุด) ตาราง Confusion Matrix จึงช่วยจำแนกดูจุดบกพร่องแยกประเภท:

🎯 Accuracy (ความแม่นยำรวม) สัดส่วนที่ทำนายประเภทได้ถูกต้องทั้งหมด $\rightarrow$ (TP + TN) / Total
🔍 Precision (ความแม่นยำคาดเดาบวก) สัดส่วนการทายว่าบวกแล้วถูกจริง เพื่อจำกัดความตื่นตูมเตือนผิดพลาด (FP) $\rightarrow$ TP / (TP + FP)
🛡️ Recall (ความสามารถตรวจเจอ) สัดส่วนคนเป็นโรคจริงที่ถูกค้นหาเจอ เพื่อจำกัดไม่ให้หลุดรอดขาดตรวจ (FN) $\rightarrow$ TP / (TP + FN)
⚖️ F1-Score: ค่าเฉลี่ยกลางแบบฮาร์มอนิก (Harmonic Mean) เพื่อประเมินความสอดคล้องโดยไม่ปล่อยให้ค่าใดค่าหนึ่งของ Precision หรือ Recall มีค่าต่ำจนเกินไป
สถานการณ์คัดกรองโรคระบาด (กำหนดกลุ่มประชากรรวมคงที่ 200 รายชิ้น)

• กลุ่มผู้ติดเชื้อจริง (Actual Positive) = บังคับคงที่ 50 ราย (คำนวณแชร์สัดส่วนระหว่าง ทายถูกตรงเป๊ะ กับ ทำหลุดรอดไป)

• กลุ่มคนปกติไม่ติดเชื้อจริง (Actual Negative) = บังคับคงที่ 150 ราย (คำนวณแชร์สัดส่วนระหว่าง ทายตื่นตูมฟรี กับ ทายถูกว่าปกติ)

PCA & K-Means Clustering

การลดมิติข้อมูลด้วย PCA และการแบ่งกลุ่มข้อมูลด้วย K-Means

🔬 PCA — การลดมิติข้อมูล

PCA ช่วย "บีบ" ข้อมูลหลาย dimension ให้เหลือน้อยมิติ แต่ยังรักษารูปแบบข้อมูลไว้

📌 Principal Component Analysis (PCA) เปลี่ยนแกนของข้อมูลให้ PC1 เก็บ "ความแปรปรวน" ไว้มากที่สุด — เหมือนหาทิศทางที่ข้อมูล "กระจาย" มากที่สุด

✅ ใช้ PCA เมื่อ...

  • มีตัวแปรเยอะ (10+ features)
  • ต้องการ visualize ใน 2D/3D
  • ตัวแปรมี multicollinearity
  • ก่อนทำ clustering

⚠️ ข้อควรระวัง

  • ต้อง standardize ข้อมูลก่อน
  • PC ไม่มีความหมาย "ตรงๆ"
  • สูญเสียข้อมูลบางส่วนเสมอ
  • ดู explained variance ratio

🎯 K-Means Clustering

K-Means แบ่งข้อมูลออกเป็น k กลุ่ม โดยจัดแต่ละจุดให้อยู่ในกลุ่มที่ centroid ใกล้ที่สุด

เตรียมความพร้อมก่อนเริ่มเรียนรู้ K-Means Clustering

Fundamentals & Workflow

Cluster (กลุ่มข้อมูล) กลุ่มของข้อมูลที่มีลักษณะคล้ายคลึงกันและอยู่ในบริเวณใกล้เคียงกัน
k (จำนวนกลุ่ม) จำนวนกลุ่มที่เราต้องการให้โมเดลแบ่ง (เช่น k = 3 คือแบ่งเป็น 3 กลุ่ม)
Centroid (จุดศูนย์กลาง) จุดศูนย์กลางของแต่ละกลุ่ม ซึ่งถูกเลือกขึ้นมาเพื่อเป็นตัวแทนของแต่ละ Cluster
📌 K-Means ทำงานอย่างไร? (อธิบายเข้าใจง่าย)
1 กำหนดจำนวนกลุ่ม (k) 2 สุ่มตำแหน่ง Centroid 3 จัดแต่ละจุดเข้ากลุ่ม 4 ขยับ Centroid ใหม่ 5 ทำซ้ำจนไม่เปลี่ยน

📊 Elbow Method: การเลือกจำนวนกลุ่ม (k) ที่เหมาะสม

ในโลกความจริง เรามักไม่รู้ว่าควรแบ่งข้อมูลเป็นกี่กลุ่มดี Elbow Method คือเทคนิคการหาค่า k ที่เหมาะสมโดยการทดลองใช้ค่า k หลาย ๆ ค่า แล้ววาดกราฟความสัมพันธ์เพื่อหาจุดหักมุมที่คล้ายกับ "ข้อศอก" (Elbow Point) โดยวิเคราะห์จากค่า WCSS (ผลรวมระยะห่างกำลังสองภายในกลุ่ม) ยิ่งค่า WCSS น้อย แสดงว่าข้อมูลในแต่ละกลุ่มยิ่งเกาะกลุ่มกันได้หนาแน่นและมีประสิทธิภาพ
