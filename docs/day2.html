คลังข้อมูล (Data Warehouse) คืออะไร?

คลังข้อมูลคือระบบจัดเก็บข้อมูลกลางที่รวบรวมข้อมูลจากหลายแหล่ง ผ่านการจัดระเบียบและทำความสะอาดแล้ว เพื่อรองรับการวิเคราะห์ รายงาน และการตัดสินใจทางธุรกิจอย่างมีประสิทธิภาพ

คุณสมบัติ 4 ประการ — คลิกพลิกการ์ดเพื่อดูรายละเอียด
🎯
Subject-Oriented
จัดตามหัวข้อธุรกิจ · คลิกพลิก

จัดระเบียบตามหัวข้อธุรกิจ เช่น ยอดขาย การเงิน สินค้าคงคลัง ไม่ใช่ตามระบบปฏิบัติการ ทำให้นักวิเคราะห์หาข้อมูลได้ง่าย

🔗
Integrated
บูรณาการจากหลายแหล่ง · คลิกพลิก

รวมข้อมูลจากหลายระบบให้เป็นรูปแบบเดียวกัน เช่น รหัสสินค้า หน่วยวัด รูปแบบวันที่ เพื่อวิเคราะห์ข้ามระบบได้ถูกต้อง

🔒
Non-Volatile
ข้อมูลมีความเสถียร · คลิกพลิก

ข้อมูลจะไม่ถูกแก้ไขบ่อยๆ เมื่อบันทึกแล้วจะคงอยู่ ทำให้การวิเคราะห์ย้อนหลังมีความแม่นยำน่าเชื่อถือ

📅
Time-Variant
เก็บข้อมูลตามเวลา · คลิกพลิก

เก็บข้อมูลประวัติศาสตร์หลายปี ช่วยวิเคราะห์แนวโน้ม เปรียบเทียบช่วงเวลา และคาดการณ์อนาคตได้

ส่วนประกอบของคลังข้อมูล — คลิกแต่ละส่วนเพื่อขยาย
จำง่ายๆ: คลังข้อมูลคือ "ห้องสมุดข้อมูลของบริษัท" ที่รวมและจัดระเบียบทุกอย่างไว้ในที่เดียว เพื่อให้ค้นหาและวิเคราะห์ได้รวดเร็ว
OLTP กับ OLAP ต่างกันอย่างไร?
รูปแบบการประมวลผลข้อมูลแบ่งเป็นสองประเภทหลัก:
  • OLTP (Online Transaction Processing) ใช้บันทึกธุรกรรมประจำวัน
  • OLAP (Online Analytical Processing) ใช้วิเคราะห์ข้อมูลเชิงลึก

⚡ OLTP — ระบบธุรกรรม

  • 💾 INSERT / UPDATE / DELETE
  • 👥 ผู้ใช้ทำธุรกรรมพร้อมกันจำนวนมาก
  • ⏱️ ตอบสนองเร็ว < 1 วินาที
  • 📦 ข้อมูลปัจจุบัน แบบละเอียด
  • 📋 Normalized Tables
  • 🏧 ATM, ร้านค้าออนไลน์, ระบบสต็อก

🔍 OLAP — ระบบวิเคราะห์

  • 📖 SELECT เท่านั้น (อ่านอย่างเดียว)
  • 👤 นักวิเคราะห์ / ผู้บริหาร
  • ⏳ Query ซับซ้อน ใช้เวลาได้นาน
  • 📚 ข้อมูลย้อนหลัง แบบสรุป
  • ⭐ Star / Snowflake Schema
  • 📊 Power BI, Tableau, Looker

โจทย์สถานการณ์ — คลิก "เฉลย" เพื่อดูคำตอบ
ขั้นตอนการเตรียมข้อมูล (Data Preparation Pipeline)
กระบวนการ 6 ขั้นตอนก่อนนำข้อมูลเข้าคลังข้อมูล — คลิกแต่ละขั้นตอนเพื่อทดลองด้วยตัวเอง
Pipeline อัตโนมัติ: กระบวนการทั้ง 6 ขั้นตอนสามารถเขียนเป็น Python script และตั้งเวลารันอัตโนมัติด้วย Apache Airflow หรือ dbt ได้
ทดสอบความเข้าใจ: ทำความสะอาดข้อมูล (Data Cleansing)
ข้อมูลดิบในโลกจริงมักมีปัญหามากมาย — ทดลองระบุและแก้ไขปัญหาด้วยตัวเอง
